BRI Bulk Insert Method


BRI Bulk Insert Method is a game changing method to insert thousands of rows of financial data in seconds. The pilot project to implement this method is "Mass Fund Transfer" feature in BRI Cash Management System (ibank.bri.co.id).

BRI Bulk Insert Method


The old method read, validate, and insert the data to the database one by one and thus takes so much time to process. In a case where a user upload a 1.000 rows of data, it takes hours to process all the data.

BRI Bulk Insert Method


BRI Bulk Insert Method process everything in batches using a lot of SQL commands. It offers an up to 60 times faster processing time for a file with more than 100 rows of data.

BRI Bulk Insert Method


BRI Bulk Insert Method was implemented in the Mass Fund Transfer using Asp.Net, NHibernate, MySQL bulk insert, Quartz.Net, and BRI Interface Service (API) technology.

BRI Anti Timeout Mechanism


BRI Anti Timeout Mechanism a simple yet groundbreaking mechanism to handle timeout events in banking transactions. The pilot project to implement this method is "Single Fund Transfer" feature in BRI Cash Management System (ibank.bri.co.id).

BRI Anti Timeout Mechanism


The old mechanism combine the process of the Journal Sequence creation and the booking transaction. In a case where a "Time Out" occurs, we have to check the bank statement manually to know if the transaction was success or not.

BRI Anti Timeout Mechanism


By separating the process of the Journal Sequence creation and the booking transaction in the BRI Interface, we can simply reverse the transaction and restart the booking process. Each transaction will only have 2 outputs, success or failure, reducing the transaction timeout rate to 0%.

BRI Anti Timeout Mechanism


BRI Anti Timeout Mechanism was implemented in the Single Fund Transfer using Asp.Net, NHibernate, Quartz.Net, and BRI Interface Service (API) technology.
